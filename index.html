Status & roadmap

Google Photos — Match footage
Google Photos has a read-only Picker API available in browsers. This app already lets you upload local images/video directly from device — which also picks from your Photos library on iOS and Android via the native file picker.

A full Google Photos OAuth integration (browse albums, import videos) requires a backend server to handle the OAuth flow securely. For now the quickest workaround is: open Google Photos → download the clip → upload it here.
Partial — local upload works now
💚
Fitbit Web API
What's available: Resting HR, sleep stages, sleep score, active zone minutes, daily steps, calories.
Why it needs a backend: Fitbit uses OAuth 2.0 with a Client Secret that must never be exposed in browser code. A tiny server (free-tier Vercel/Render function) handles the token exchange and relays data to this app.
To set up manually right now:
1. dev.fitbit.com → Create app → get Client ID & Secret
2. Complete OAuth at fitbit.com/oauth2/authorize
3. Exchange code for token at api.fitbit.com/oauth2/token
4. Call endpoints like /1/user/-/sleep/date/today.json
Needs backend — setup guide above
🥗
MacroFactor — Nutrition sync
MacroFactor does not have a public API as of 2025 — it's a closed platform with no third-party integration support.

Best workarounds today:
1. Manual entry — use the food log in this app's Nutrition tab to mirror your MacroFactor intake
2. CSV export — MacroFactor supports data export from Settings. If you export a CSV, you can upload it here and the AI Coach can analyse your weekly nutrition patterns
3. Watch for API release — MacroFactor has mentioned developer access on their roadmap
No public API — use manual log or CSV export
